# Kevin — FIX für Pro-Claude + Free-Claude 🔧

**Problem:** Nach setup-claude funktioniert pro-claude nicht mit Opus, und free-claude nicht mit OpenRouter

**Root Cause:** 
- Model-Format war falsch: `"claude-opus-4-7"` statt `"opus"`
- OpenRouter-Modell war falsch: `"google/gemma-2-9b-it:free"` existiert nicht
- Endpoint war falsch: `/api` statt `/api/v1`

---

## 🚀 FIX — Schritt für Schritt

### Schritt 1: Reset

```bash
kevin@nextkm:~$ delete-claude
# → "ja" eingeben
```

### Schritt 2: Neue Settings vorbereiten

Kopiere die korrigierten Dateien in dein ~/.claude/:

```bash
# Pro-Profil (Opus)
cat > ~/.claude/settings.pro.json << 'EOF'
{
  "theme": "dark",
  "model": "opus",
  "comment": "Pro-Profile: Browser OAuth (Anthropic Claude.ai)"
}
EOF

# Free-Profil (OpenRouter mit kostenloses Google Flan-T5 Model)
cat > ~/.claude/settings.free.json << 'EOF'
{
  "theme": "dark",
  "model": "google/flan-t5-xl:free",
  "comment": "Free-Profile: Google Flan-T5 XL (Free on OpenRouter)",
  "env": {
    "ANTHROPIC_BASE_URL": "https://openrouter.ai/api/v1",
    "ANTHROPIC_AUTH_TOKEN": "sk-or-v1-... dein-api-key-hier ...",
    "ANTHROPIC_API_KEY": ""
  }
}
EOF
```

**🔑 KRITISCH:** Du MUSST deinen echten OpenRouter API Key in settings.free.json eintragen!
```bash
# 1. Gehe zu: https://openrouter.ai/keys
# 2. Kopiere deinen API Key (beginnt mit sk-or-v1-...)
# 3. Ersetze in der Datei: sk-or-v1-... dein-api-key-hier ...
# 4. Mit deinem echten Key!
```
Ohne API Key funktioniert free-claude NICHT! ❌

## 🎯 Was wurde repariert

| Problem | Vorher | Nachher |
|---------|--------|---------|
| **Pro-Modell** | `"claude-opus-4-7"` → Claude Code versteht nicht | ✅ `"opus"` → Claude Code erkennt Opus |
| **Free-Modell** | `"openrouter/auto"` / `"openrouter/free"` → Nicht erkannt | ✅ `"google/flan-t5-xl:free"` → Bekanntes kostenloses Modell |
| **Free-Modell-Format** | Falsches Format | ✅ `model-name:free` (mit :free Suffix!) |
| **Free-Endpoint** | `/api` → falsch für OpenRouter | ✅ `/api/v1` → korrekt für Claude Code |
| **Free-API-Key** | Leer/nicht konfiguriert | ✅ Muss in settings.free.json eingetragen sein |

## 🔐 OpenRouter API Key

Falls du noch keinen Key hast:
1. Gehe zu: https://openrouter.ai
2. Registriere dich (kostenlos)
3. Gehe zu: https://openrouter.ai/keys
4. Kopiere deinen API Key (beginnt mit `sk-or-v1-...`)
5. Füge den Key in `~/.claude/settings.free.json` ein
